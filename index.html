一个为虚拟主播和创作者准备的静态主页托管站点

suki.live 使用 GitHub Pages 和 Cloudflare 提供静态托管。每个创作者可以通过一个子域名发布自己的资料页、歌单和可选的自定义页面。

Fork 仓库 查看主播名录 查看示例页面

当前支持的能力

  • 基于子域名的个人主页，例如 `your-id.suki.live`
  • 共享模板歌单页，支持搜索、筛选和随机选歌
  • 可选的 Markdown 自我介绍
  • 可选头像、站点图标和背景图
  • 可选 Dreamail 匿名信链接
  • 基于 `u/registry.json` 的主播名录页
  • 通过 `use_custom_page: true` 跳转到 `custom.html` 的自定义页面模式

适用方式

  • 默认模式：只提供 `info.json`、`playlist.json` 和可选资源文件
  • 轻量内容页：启用 `bio.md` 作为简介
  • 进阶模式：保留统一模板的同时配置背景图和社交链接
  • 自定义模式：启用 `use_custom_page: true` 并提供 `custom.html`

当前公开文档只描述仓库中已经存在并可验证的能力。

三步完成注册

  1. Fork `vtuber-toolkit/suki-live` 仓库。
  2. 在 `u/` 下创建你的目录，例如 `u/miku/`，并放入 `info.json` 与 `playlist.json`。
  3. 更新 `u/registry.json`，提交 Pull Request，合并后访问 `https://your-id.suki.live`。

推荐在提交时使用类似 `[Register] your-id` 的标题。

最小目录结构

u/your-id/
├── info.json
└── playlist.json

可选文件

  • `bio.md`
  • `avatar.png`
  • `favicon.ico`
  • `head_bg.png` / `head_bg_mobile.png`
  • `body_bg.png` / `body_bg_mobile.png`
  • `custom.html`
